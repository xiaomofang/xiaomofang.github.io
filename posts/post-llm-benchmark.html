一次大模型推理性能压测的完整记录

压测不是为了得到一个好看的数字，而是为了回答一个问题： 在当前硬件和框架下，系统的瓶颈在哪里？

核心指标

TTFT（首 token 延迟）、TPOT（每 token 耗时）、吞吐（tokens/s） 三者往往此消彼长，需要在固定场景下同时观测。

场景设计

python bench_serve.py --concurrency 1,4,8,16 --input-len 512 --output-len 128

一个更深的追问

当并发升高、吞吐上升时，P99 延迟恶化了多少？ 这个 trade-off 是否可接受——这才是压测真正要回答的问题。
